## ÔøΩ Deployment

### CI/CD Pipeline Status
**‚úÖ CI/CD pipelines are working and operational**

**Backend Deployment:**
1. **Automated CI/CD**: Push to main ‚Üí Triggers build, test, and Docker image creation
2. **Manual Production Deploy**: Due to Oracle free tier IP restrictions, backend deployment uses manual trigger:
   ```bash
   fly deploy
   ```
   This approach:
   - ‚úÖ Uses your whitelisted IP during local Docker build process
   - ‚úÖ Deploys via `fly.toml` configuration with `Dockerfile` and `start.sh` 
   - ‚úÖ No dependency on external Docker registry pulls
   - ‚úÖ Reliable deployment to: https://inventoryservice.fly.dev

**Frontend Deployment:**
- ‚úÖ **Fully Automated**: Push to main ‚Üí Auto-build and deploy to Koyeb
- ‚úÖ **Live Application**: https://inventory-service.koyeb.app
- ‚úÖ **Health Checks**: Automated smoke tests and health monitoring

### Why This Deployment Strategy Works
- **Oracle Free Tier**: Requires IP whitelisting for database connections
- **Fixed IP Solution**: Using Fly.io VM provides stable IP address
- **Local Build**: Ensures build process uses your whitelisted IP
- **No Daily IP Updates**: Eliminates need to constantly update Oracle IP whitelist

### Production Environment
- **Backend**: Spring Boot with `application-prod.yml` profile
- **Database**: Oracle Autonomous DB with wallet-based authentication
- **Security**: OAuth2 with Google integration
- **Monitoring**: Health check endpoints and application logging

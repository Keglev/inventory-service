Architecture Decision Records (ADRs)

This directory contains architecture decisions with justification.

ADRs answer: - Why we chose a design (and why not alternatives) - what tradeoffs we accept - where the decision is implemented (high-level pointers)

ADRs do not document: - how a component works internally - step-by-step implementation details

Index

  • ADR-0003: DTO boundary — no entity exposure across layers
  • ADR-0004: HTTP status as the error envelope (no success wrapper)
  • ADR-0005: Error message sanitization in GlobalExceptionHandler
  • ADR-0006: Custom repository implementations for dialect-aware analytics queries
  • ADR-0007: Cross-origin session cookie configuration for split frontend/backend deployment
